# Task-001 Analyst Instruction

## Goal
実装者が迷わない要件定義を作る。

## Steps
1. ユースケースを5つ以上定義する。
2. 環境判定ルールを優先順付きで記述する。
3. 受け入れ条件をテスト可能な文で定義する。
4. 非機能要件（性能、保守性、可観測性）を整理する。

## Output
`analyst/deliverables/task-001-requirements.md`

## Required Sections
- Scope in / out
- User stories
- Acceptance criteria
- Open questions
