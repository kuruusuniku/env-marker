# Task-004 QA Instruction

## Goal
リリース前検証を実施し、客観的な検証証跡を提出する。

## Checklist
- 主要機能の回帰確認
- インストール/更新/削除手順確認
- 既知不具合の再現確認
- Web Store提出物の妥当性確認

## Steps
1. リリース候補ビルドを固定し、検証対象を明示する。
2. チェックリストを実行し、結果を記録する。
3. Blocker/Critical の未解決項目を抽出する。
4. `devils-advocate` が判断できる形式で証跡をまとめる。

## Output
`qa/deliverables/task-004-release-checklist.md`

## Required Sections
- Build/Version Under Test
- Verification Results
- Open Defects
- Release Risks
